# Simulation Guide: Real-World Ingestion & AI Evolution

This guide helps you simulate a maturing data pipeline in your Semantic Layer.

## Step 1: Baseline Ingestion
**File:** `simulation/batch_1_standard.csv`
**What to expect:**
- Standard mapping of accounts, contacts, and deals.
- All fields should map deterministically without AI assistance.
- **Verification:** Check `silver.dim_account` for Acme Corp and GloboLink.

## Step 2: Scale Ingestion
**File:** `simulation/batch_2_standard.csv`
**What to expect:**
- More entities, consistent schema.
- One deal for **InGen** is marked as `Closed Won`.
- **Verification:** Check `gold.revenue_summary` to see total won value per account.

## Step 3: Schema Evolution (Simple)
**File:** `simulation/batch_3_evolution.csv`
**What to expect:**
- This file contains a new column: **`lead_source`**.
- The AI will detect it, classify it into `dim_account`, and add it to the DB.
- Logs: `🤖 Unknown field 'lead_source' -> asking AI...`

## Step 3b: Dynamic Schema Awareness (Continuation)
**File:** `simulation/batch_3_continuation.csv`
**What to expect:**
- This file has the same **`lead_source`** column from Step 3.
- **Result:** The system should now recognize `lead_source` as a "known" column because it was already added to the database.
- **Critical:** You SHOULD NOT see the AI logs again for this field. It should be handled deterministically.

## Step 4: Complex Schema Evolution
**File:** `simulation/batch_4_complex_evolution.csv`
**What to expect:**
- This file introduces **two** new columns: `deal_priority` and `preferred_contact_time`.
- AI must decide: `deal_priority` -> `fact_deals`, `preferred_contact_time` -> `dim_contact`.
- **Result:** Multiple tables will evolve simultaneously.

## Sample Queries for Testing (Run in pgAdmin)

### 💿 Silver Layer Query (Raw Entities)
```sql
-- See all evolved data in the account dimension
SELECT account_name, industry, geography, lead_source FROM silver.dim_account ORDER BY created_at DESC;

-- See specific deal details with owner and stage
SELECT deal_name, account_name, deal_value, deal_stage, owner FROM silver.fact_deals ORDER BY created_at DESC;
```

### 🥇 Gold Layer Query (Business Answers)
```sql
-- Business Question: "Which accounts have the most revenue exposure?"
SELECT account_name, industry, total_deal_value, won_value, pipeline_value FROM gold.revenue_summary ORDER BY total_deal_value DESC;

-- Business Question: "Show me a 360 view of my top accounts"
SELECT * FROM gold.account_360 WHERE total_deal_value > 0 ORDER BY total_deal_value DESC;
```
